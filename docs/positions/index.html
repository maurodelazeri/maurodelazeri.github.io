Positions

#Positions

Note

Position accounting depends on a trading account

A position is a trade obligation, i.e. the number of bought or sold contracts of a financial instrument. A long position is financial security bought expecting the security price go higher. A short position is an obligation to supply a security expecting the price will fall in future.

The platform allows you to easily track how a position was opened or how a deal was performed. Each trading operation has its unique ID called a position_id. Each order and deal receive a order_id relating to their relevant position. Each deal receives a deal_id of an order, by which it was concluded.

If a position was affected by multiple deals, for example in the case of a partial closing or increasing volumes, each of the deals feature the position's position_id. This makes it easy to track the entire history of the position as a whole.

If trading operations are sent to an exchange or a liquidity provider, they additionally feature an ID from an external system. This allows additional tracking of the interrelation of operations away from the platform.

#GetPositions()

Returns an position

Parameters:

In order to request the position, you have to specify the the field position_id on your PositionRequest. Any additional parameters on the PositionRequest will be ignored.

The request requires a structure OrderRequest with the following fields:

#PositionRequest

fieldtypedefaultrequireddescription
position_idstringNULLtrueposition id

Return Value

In case of a successful basic check of protobuf message PositionResponse retcode. However, this is not a sign of successful execution of a position operation. For a more detailed description of the function execution result, analyze the fields of result protobuf message.
