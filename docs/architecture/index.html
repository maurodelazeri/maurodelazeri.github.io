Architecture

#Architecture

Zinnion architecture is divided in three main components:

#User Infraestructure

This part is completely private, the user can use its current infrastructure or just its computer and leverage Zinnion using a binary called Shark, this binary provides all the functionality needed to build an algorithm like simulation, reports, live and historical data and much more. The bottom line is that the user only needs to focus on developing the core logic of their algorithm and everything else we do for you so that you can focus on what matters most.

We also support a research mode in which you can take advantage of the most popular notebooks for python, R and Matlab and run statistical models against our historical database.

Your algorithm communicates with Shark through a binary wire protocol gRPC, we currently support the following programing languages : C / C++ / C# / Julia / Matlab / F / R / OCaml / Haskell / Erlang / Elixir / Python / Dart / Go / Java / Scala / Objective-C / PHP / Ruby / WebJS

#Zinnion Infraestructure

Zinnion offers a full range of tools that add extra functionality to your algorithm without having to code them such as a complete web interface to analyze what your algorithm is doing and tools to systematize the simulation and find the best parameters you can use to achieve the best results.

#External Services Infraestructure

Right out of the gate you have connection with more than 15+ venues with standardized live data feeds and execution. You can execute and manage positions with one line of code.
